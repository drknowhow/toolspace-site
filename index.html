toolspace

A registry of install manifests for autonomous agents.

Toolspace hosts install-manifest-spec manifests — signed JSON descriptions of what an agent needs to set up a tool: env vars, prompts, verification, revocation. Agents can install from this registry, from raw GitHub URLs, or from any mirror — the spec is the moat, not the registry.

Get started

pip install install-manifest
install-manifest install <url-or-path>

What a manifest looks like 

{
  "schema_version": "0.2",
  "name": "gmail",
  "version": "1.0.0",
  "description": "Send and search a Gmail mailbox.",
  "auth":    { "kind": "oauth2", "...": "..." },
  "actions": [
    { "name": "send", "input_schema": { "...": "..." }, "side_effects": "external" }
  ]
}

Full reference: /examples/gmail.v0.2.json. Every field is documented in the v0.2 schema.

For agents

Schema v0.2 · stable v0.1 · legacy

Current: install-manifest-v0.2.json — adds the structured actions[] catalog so agents can invoke each operation a tool exposes with typed input/output and side-effects classification. CORS-enabled, cached.

Legacy install-manifest-v0.1.json remains served unchanged for tools targeting the original MCP-shaped format. New manifests should target v0.2.

Manifest index

/manifests.json — list of curated manifests with capability tags. The versions[] array lists every supported schema version.

Example manifest

/examples/gmail.v0.2.json — reference v0.2 manifest with five Gmail actions (whoami, search, send, label_thread, unlabel_thread), each with typed input/output and side-effects classification. Use this as the template when authoring new manifests.

A v0.1 example is also available for tools still targeting the legacy format.

For humans

You're welcome to read along. The spec is open and the registry is public. If you build agents and want yours discoverable here, open a PR on toolspace-site with your manifest URL added to manifests.json.

CLI 

pip install install-manifest
install-manifest validate <url-or-path>
install-manifest install <url-or-path>

The install-manifest CLI ships with the schema bundled, so it works offline. Network is only used for the manifest itself.
